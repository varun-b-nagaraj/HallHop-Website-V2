# HallHop Design System

## Brand Positioning

HallHop is a premium education technology platform focused on hallway accountability, student safety, and real-time school operations. The website should feel modern, trustworthy, and sophisticated — closer to a polished SaaS/security product than a student club website.

Core message:

> Smarter Passes. Safer Schools.

Supporting message:

> HallHop replaces paper hall passes with a real-time digital system that helps schools track student movement, reduce misuse, and make faster, more informed decisions.

## Visual Direction

The design should feel:
- Clean
- Premium
- Modern
- Calm
- Secure
- Data-driven
- Sophisticated
- Built for administrators, teachers, and school districts

Avoid:
- Childish school visuals
- Random stock-photo overload
- Bright primary-school colors
- Cluttered layouts
- Generic Wix-style sections
- Excessive gradients
- Cartoonish icons

## Color Palette

Use a restrained, premium palette.

### Primary

- Near Black: `#080B12`
- Deep Navy: `#0B1220`
- Slate Navy: `#111827`
- Pure White: `#FFFFFF`
- Off White: `#F8FAFC`

### Accent

- HallHop Blue: `#2563EB`
- Electric Blue: `#3B82F6`
- Soft Blue: `#DBEAFE`
- Cyan Accent: `#06B6D4`

### Neutrals

- Slate 900: `#0F172A`
- Slate 700: `#334155`
- Slate 500: `#64748B`
- Slate 300: `#CBD5E1`
- Slate 100: `#F1F5F9`

### Usage

- Use white/off-white backgrounds for most sections.
- Use deep navy/near-black for premium contrast sections.
- Use blue accents sparingly for CTAs, icons, highlights, and active states.
- Avoid large flat blue sections unless balanced with whitespace.

## Typography

Use a modern sans-serif.

Recommended:
- `Inter`
- `Geist`
- `Satoshi`
- `Manrope`

Hierarchy:

### Hero headline
- 56–76px desktop
- 40–48px tablet
- 34–40px mobile
- Font weight: 700–800
- Tight line height

### Section headings
- 36–48px desktop
- 28–36px mobile
- Font weight: 700

### Body text
- 16–18px
- Line height: 1.6–1.8
- Color: slate gray, not pure black

### Labels / eyebrow text
- 12–14px
- Uppercase or semi-uppercase
- Letter spacing: 0.08em
- Blue or slate

## Layout Principles

Use a premium SaaS layout.

Rules:
- Max content width: 1120–1240px
- Strong vertical spacing
- Generous whitespace
- Sections should breathe
- Avoid dense paragraphs
- Break long copy into cards, bullets, and short statements
- Use 12-column grid behavior on desktop
- Stack sections naturally on mobile

Recommended section spacing:
- Desktop: 96–140px vertical padding
- Mobile: 64–88px vertical padding

## Navbar

Navbar should be:
- Sticky
- Transparent at top
- Blurred white background after scroll
- Subtle border/shadow after scroll
- Logo left
- Links center or right
- CTA button right
- Mobile hamburger menu

Links:
- Home
- Our Story
- Problems
- Solutions
- Projects
- Team
- Contact

CTA:
- “Contact Us”
- or “Explore Features”

## Hero Section

Hero should be the strongest section.

Content:
- Eyebrow: `Digital Hall Pass System`
- Headline: `Smarter Passes. Safer Schools.`
- Subheadline: `HallHop replaces paper passes with real-time digital tracking, schedule-aware workflows, and actionable insights for safer, more accountable schools.`
- Primary CTA: `Explore Features`
- Secondary CTA: `See How It Works`

Visual:
- Create a polished dashboard mockup.
- Include active passes, student status, destination, timer, and alerts.
- Add a Chrome Extension mini-card beside it.
- Use shadows, rounded corners, subtle gradients, and floating animation.

## Cards

Cards should use:
- Rounded corners: 20–28px
- Subtle border: `1px solid rgba(15, 23, 42, 0.08)`
- Background: white or translucent white
- Shadow: soft, not harsh
- Hover: slight lift, border color shift, subtle shadow increase

Card style:
- Icon top
- Short title
- 2–3 sentence description max
- Optional small metadata label

## Buttons

Primary button:
- Blue background
- White text
- Rounded full or 14px
- Medium shadow
- Hover: slightly darker blue and lift

Secondary button:
- White/transparent background
- Slate text
- Border
- Hover: light slate background

Button padding:
- Desktop: 12–18px vertical, 20–28px horizontal
- Mobile: full-width where appropriate

## Animation

Use animations tastefully.

Recommended:
- Hero text fades/slides upward on load
- Cards fade up on scroll
- Dashboard mockup floats slowly
- Navbar transitions on scroll
- Icons gently scale or shift on hover
- CTA section has subtle gradient motion

Timing:
- 200ms for hover
- 400–700ms for section entrance
- Use easing: `easeOut`, `easeInOut`

Avoid:
- Bouncy animations
- Spinning icons
- Too many simultaneous effects
- Slow animations that delay usability

## Imagery

Use minimal imagery.

Preferred:
- Abstract hallway gradients
- Dashboard mockups
- Interface visuals
- Subtle blurred school corridor backgrounds
- Security/data visuals

Avoid:
- Too many student stock photos
- Low-resolution images
- Obvious AI images
- Watermarked assets
- Random unrelated graphics
